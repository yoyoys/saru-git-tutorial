建立Pull Request

Backlog篇

從Git數據庫一覽中選擇目標數據庫，然後開啟分支一覽。

可以看到剛才建立的「add-sort-func」分支。請點擊Pull Request欄位的「追加」。

會顯示以下畫面。

  1. 目標分支
    要合併Pull Request的目標分支。
  2. Pull Request分支
    要被合併的分支。
  3. Pull Request負責人
    負責審核和合併所建立的Pull Request的負責人。

輸入適當的內容後，點擊「追加Pull Request」按鈕來建立Pull Request。建立Pull Request後，會通知Pull Request負責人和想要通知的使用者。建立後會顯示以下畫面，可以確認內容。

GitHub篇

點擊目標數據庫的「Compare & pull request」。

會顯示以下畫面。

  1. 目標分支
    要合併Pull Request的目標分支。
  2. Pull Request分支
    要被合併的分支。

輸入適當的內容後，點擊「Create pull request」按鈕來建立Pull Request。建立後會顯示以下畫面，可以確認內容。
